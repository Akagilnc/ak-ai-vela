## Vision (long-term north star, NOT MVP scope)

### 10x Check
The 10x version is a **living dashboard** where gap analysis updates as families add data over semesters. Not "use once and forget," but "check every semester." This is the true fulfillment of the "replace agency long-term tracking" promise.

### Platonic Ideal
Mom opens the tool, fills in child info in 3 minutes. She sees her child's profile overlaid on target school admission profiles. She can toggle "what if my child takes AP Physics?" and see gaps change. Every recommendation is actionable. She screenshots and shares to her WeChat group.

*This section describes the long-term direction. MVP scope is defined below.*

## Seed User Context
**Target:** Undergraduate application (pre-vet / animal science / biology track) at top 30 US universities.
The seed user's child wants to pursue veterinary medicine. In the US, most vet programs (DVM) are graduate-level. Undergrad students typically major in animal science, biology, or a pre-vet track. Key school filter: top 30 universities WITH strong pre-vet or animal science programs (e.g., Cornell, UC Davis, Colorado State, Wisconsin-Madison). This shapes school list selection, gap analysis dimensions (science GPA, animal-related experience), and questionnaire design (science coursework, animal experience hours).

## Product Framing Decision
Vela is a **decision support system**, not just an information tool. Parents lack confidence, not just information. Reports must give clear, actionable recommendations ("you should do X"), not just display data and gaps.

## Implementation Approach
**Full-stack local deployment:** Next.js + TypeScript + Tailwind + Prisma/SQLite. School database + questionnaire + interactive assessment report + growth portfolio DB schema. LLM is backend-only tool for founder to polish report content. Local deployment on founder's machine.

*This supersedes the design doc's "long image / PDF export" requirement. Interactive report screenshot via html2canvas is the MVP export mechanism.*

## Full MVP Scope (design doc + accepted expansions)
1. **Data layer:** MVP ships with 10-15 core schools aligned with seed user's target: top 30 universities with strong pre-vet/animal science programs. 50 schools is a Phase 2 target, not MVP. CDS data with international student fields (international acceptance rate, international scholarships, visa/OPT support, international student %, English requirements). School records should include a `programs` field to tag pre-vet/animal science/biology availability.
2. **Input layer:** Static parent questionnaire based on design doc's initial draft (8 sections: child basics, school system, academic performance, extracurriculars, special experiences, family finances, target preferences, biggest concern). Add seed-user-relevant fields: target major direction (pre-vet/animal science/biology/other), science course grades if available, animal-related experience (type, hours, duration). To be revised after co-designer conversation. Reference: design doc section "初版问卷". Open text field "biggest concern" limited to 200 characters.
3. **Analysis layer:** Rule-driven gap analysis comparing student profile fields against school admission profile fields. Gap scoring model (implementation task): for each dimension, compute distance between student value and school target range, output as gap severity (green/yellow/red). Supports dynamic input for "What If" recalculation on numeric/count fields. Pure deterministic logic, no LLM.
4. **Output layer:** Interactive assessment report. Each gap is clickable → shows current value, target range, explanation, actionable recommendation. Focus on clear "you should do X" recommendations, not parameter exploration.
5. **Browse layer:** School database with filtering, comparison, radar charts. All radar dimensions normalized to 0-100 percentile scale. Configurable dimension list.
6. **Growth layer:** Database schema only. Table: `questionnaire_results` with columns (id, student_id, submitted_at, answers JSON). No comparison UI in MVP. Schema enables Phase 2 growth tracking without data migration.
7. **Export layer:** Screenshot of interactive report for WeChat sharing. Uses html2canvas. Not a separate static report — it's a snapshot of the interactive page.
8. **UX layer:** Bilingual glossary tooltips (~20-30 terms), Chinese-first interface.
9. **Backend:** LLM as founder's tool for polishing report recommendation text. Founder reviews and approves every report before delivery. LLM never involved in real-time user interactions or "What If" calculations.

## Data Entry Strategy
MVP: 10-15 schools manually curated by founder from CDS PDFs/websites. ~150-225 data points. Estimated effort: 3-4 hours manual work (not code work). Founder does this.

Phase 2: Expand to 50 schools. Consider semi-automated CDS PDF extraction at that point.

Data entry template: a JSON or CSV file per school, validated by Prisma schema on import. Build a simple seed script.

## GPA Normalization
Chinese school GPA (percentage or rank-based) cannot directly map to US 4.0 scale.

MVP mapping table:
| Chinese input | Normalized GPA | Percentile |
|--------------|----------------|------------|
| Top 5% or 95+ points | 3.9+ | 95th |
| Top 10% or 90-94 points | 3.7-3.9 | 90th |
| Top 25% or 85-89 points | 3.5-3.7 | 75th |
| Top 50% or 80-84 points | 3.0-3.5 | 50th |
| Below 50% or <80 points | <3.0 | <50th |

Edge cases:
- Student reports percentage score (e.g., 85/100): map directly to the points column
- Student reports class rank (e.g., 5th out of 200): compute percentile, map to GPA
- School system is non-standard or data insufficient: display "暂无可比数据" and skip GPA gap
- Never fabricate a normalized score. Show "needs more info" rather than guess.

This is a starting approximation. Refine after co-designer feedback.
